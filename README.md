# 🎓 Software Engineering & Systems Mastery Encyclopedia

> **"From Pointers to Distributed Systems: A Hardcore Engineering Journey"**

Esta no es una simple colección de ejercicios. Es una enciclopedia práctica y progresiva diseñada para formar ingenieros de software de élite, arquitectos de sistemas y programadores de bajo nivel. Utilizando exclusivamente **C (C17+)**, **C++ (C++23)** y **Rust (Stable)**, este repositorio cubre desde los fundamentos de la computación hasta el desarrollo de kernels y sistemas distribuidos.

---

## 🗺️ The Ultimate Learning Roadmap

La secuencia pedagógica está diseñada como una carrera universitaria intensiva combinada con experiencia profesional real.

### 🏛️ Fase I: Fundamentos de Hierro (Nivel 00-02)
*   **00 - CS Fundamentals**: Historia, arquitectura de Von Neumann, CPU, RAM y jerarquía de caché.
*   **01 - Programming Fundamentals**: Lógica booleana avanzada, recursión, modularización y paradigmas.
*   **02 - Memory & Architecture**: Virtual memory, stack vs heap, segmentación, y alineación de memoria.

### 🛠️ Fase II: El Triunvirato de Sistemas (Nivel 03-05)
*   **03 - C (The Foundation)**: Manual memory management, punteros a funciones, allocators personalizados y POSIX.
*   **04 - C++ (The Power)**: Template Metaprogramming, Concepts, Coroutines y Move Semantics (C++23).
*   **05 - Rust (The Future)**: Ownership, Borrowing, Async/Tokio, FFI y Unsafe Rust.

### 🧬 Fase III: Estructuras y Algoritmos (Nivel 06-07)
*   **06 - Data Structures**: Implementación desde cero de árboles (AVL, Red-Black, B+), Hash Tables y Skip Lists.
*   **07 - Algorithms**: Grafos avanzados (A*, Floyd-Warshall), Programación Dinámica y Backtracking.

### 🏗️ Fase IV: Ingeniería y Arquitectura (Nivel 08-11)
*   **08/09 - Debugging & Testing**: Fuzzing, Property-based testing y Profiling avanzado.
*   **10/11 - Design & SE**: SOLID, Clean Architecture, DDD y patrones de diseño en lenguajes de sistemas.

### ⚙️ Fase V: Sistemas Operativos y Bajo Nivel (Nivel 12-13, 19, 23)
*   **12 - Operating Systems**: Scheduler, Paging, Context Switching y File Systems.
*   **13 - Concurrency**: Lock-free y Wait-free programming, Atomics y Memory Barriers.
*   **19/23 - Linux & Kernel**: eBPF, Drivers, Interrupts y System Calls.

### 🌐 Fase VI: Infraestructura y Redes (Nivel 14-15, 22)
*   **14 - Networking**: TCP/UDP, TLS 1.3, QUIC y WebSockets desde cero.
*   **15 - Databases**: Motores SQL, índices B-Tree y Query Planners.
*   **22 - Distributed Systems**: Raft, Paxos, Consistent Hashing y Sharding.

## 📂 Estructura de cada Módulo

Cada tema dentro de los directorios sigue este estándar riguroso:

1.  **`README.md`**: Introducción y guía rápida.
2.  **`theory.md`**: Explicación profunda (Qué, Por qué, Cuándo, Complejidad, Referencias Académicas).
3.  **`examples.md`**: Código en C, C++ y Rust comparando implementaciones.
4.  **`exercises.md`**: Problemas de nivel básico a "hardcore" (tipo entrevista FAANG).
5.  **`solutions.md`**: Explicación detallada de las soluciones.
6.  **`references.md`**: Papers, libros (como el *Dragon Book* o *TAOCP*) y documentación oficial.

## 🚀 Cómo usar este repositorio

1.  **Sigue el orden numérico**: Los conceptos se construyen unos sobre otros.
2.  **No te saltes la teoría**: Entender el *porqué* es lo que diferencia a un programador de un ingeniero.
3.  **Compila y experimenta**: Cada módulo incluye archivos de build (Make/CMake/Cargo).
4.  **Benchmarking**: Usa las herramientas incluidas para medir el impacto de tus optimizaciones.
